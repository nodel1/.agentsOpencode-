## Concepts Learned

### 1. Dynamic Discovery
- Auto-scan folders to detect available agents
- Orchestrator scans before routing

### 2. Orchestration Patterns
- Sequential execution (A → B → C)
- Parallel processing (A, B, C simultaneously)
- Pipeline patterns (output feeds next)
- Hierarchical delegation (split into simpler subtasks)

### 3. Agent Routing
- **Rule-based:** Simple keywords ("commit" → git)
- **Intent-based:** By user goal
- **Semantic:** By meaning, not words
- **LLM-based:** Complex reasoning for routing

### 4. Task Decomposition (To-Do Lists)
From Kenneth Leung's article:
- **Todo item:** `{content: string, status: pending|in_progress|completed}`
- **State:** List of todos for tracking
- **Tool:** write_todos to update list
- **System prompt:** Guides agent when to use tool

### 5. Planning Agent Components
```
1. ANALYZE → Understand user's request
2. DISCOVER → Scan available agents
3. PLAN → Split into tasks with dependencies
4. ROUTE → Select correct agent
5. EXECUTE → Execute
6. AGGREGATE → Collect results
```
