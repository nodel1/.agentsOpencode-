## Conceptos Aprendidos

### 1. Dynamic Discovery
- Auto-scan de carpetas para detectar agentes disponibles
- El orquestador escanea antes de rutear

### 2. Orchestration Patterns
- Sequential execution (A → B → C)
- Parallel processing (A, B, C simultáneamente)
- Pipeline patterns (output de uno alimenta al siguiente)
- Hierarchical delegation (dividir en subtasks más simples)

### 3. Agent Routing
- **Rule-based:** Keywords simples ("commit" → git)
- **Intent-based:** Por objetivo del usuario
- **Semantic:** Por significado, no palabras
- **LLM-based:** Razonamiento complejo para routing

### 4. Task Decomposition (To-Do Lists)
Del artículo de Kenneth Leung:
- **Todo item:** `{content: string, status: pending|in_progress|completed}`
- **State:** Lista de todos items para tracking
- **Tool:** write_todos para actualizar la lista
- **System prompt:** Guía al agente cuándo usar la tool

### 5. Planning Agent Components
```
1. ANALYZE → Entender qué pide el usuario
2. DISCOVER → Escanear agentes disponibles
3. PLAN → Dividir en tasks con dependencias
4. ROUTE → Seleccionar agente correcto
5. EXECUTE → Ejecutar
6. AGGREGATE → Recoger resultados
```
